# Clipping Ops Cockpit v0.2.0

## Highlights

- MiniMax/Hermes web cockpit is the supported operator surface; the old native Swift app path is no longer part of the shipped repo.
- Campaign review factory targets 24 review kits per day across three active streamer campaigns, with freshness recovery when the newest windows run dry.
- Review approval now auto-creates a publish package and schedules a dry-run Upload-Post validation job into the next future `:14` slot.
- Publish schedule defaults to eight local slots per day: `00:14`, `03:14`, `06:14`, `09:14`, `12:14`, `15:14`, `18:14`, and `21:14`.
- Future scheduled dry-runs are rebalanced to avoid adjacent same-streamer slots when another approved campaign is available.
- Live posting remains locked behind Upload-Post key readiness, account warm-up, live mode, and final GUI confirmation.

## Verification

- Backend smoke coverage includes auto-slot idempotency, 48-slot backlog scheduling, streamer rebalance, due tick promotion, and live-post blockers.
- Web cockpit surfaces the scheduled publish slot on review cards, detail panels, and publish status controls.
- Hermes installer includes a no-agent publish schedule tick so due slots can promote themselves without manual Codex supervision.
