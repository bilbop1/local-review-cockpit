# Clipping Ops Cockpit v0.3.0

Release date: 2026-06-21

## Executive Summary

v0.3.0 is the buddy-install release. A non-technical operator should be able to give Codex the GitHub link plus one install-mode prompt, then let Codex clone, verify, configure local secrets, lock the Upload-Post profile, install Hermes jobs, and queue the first campaign research/build wave.

This release also ships the current TikTok-only approved-post lane: approved review kits schedule into future local `:14` slots, and live posting stays locked unless the local Upload-Post key, exact profile, warmed TikTok status, live mode, and local auto-post/manual confirmation all pass.

## Give Codex This Starting Command

```text
Get into Clipping Ops installation mode for this repo:
https://github.com/bilbop1/local-review-cockpit

Assume I am not technical. Clone the repo, then read AGENT_START_HERE.md and docs/codex-buddy-bootstrap.md before changing anything. Explain in plain English what you need, ask me for one thing at a time, and do not ask me to edit files manually unless there is no safer option.

After the no-key clone check passes, run ./script/codex_buddy_bootstrap.sh and guide me through the prompts. Ask me for: MiniMax API key, Twitch client ID and secret, Kick client ID and secret if I have Kick, Upload-Post API key, exact Upload-Post profile name, whether TikTok is warmed, and whether to turn on approved-kit auto-posting. If I do not know where to find one of those, walk me through it in the browser.

Never print, commit, or store secrets in repo files. Store secrets only through the provided local scripts/Keychain flow. Keep posting TikTok-only unless I explicitly say another platform is warmed. When setup is done, verify http://127.0.0.1:8765/app works, queue starter campaign research/build jobs, and tell me to review first kits at http://127.0.0.1:8765/app/reviews in about 45-90 minutes.
```

## Major Changes

- Adds `docs/codex-buddy-bootstrap.md`, the friend-facing setup manual.
- Adds `script/codex_buddy_bootstrap.sh`, the guided install rail for new local operators.
- Adds `script/queue_buddy_campaign_kickoff.py`, a Hermes ledger starter queue for active campaigns.
- Updates README, handoff docs, command cookbook, and operator docs around profile-locked Upload-Post setup and TikTok-only default posting.
- Bumps app/backend metadata to `0.3.0` and backend API version to `2026-06-21-buddy-bootstrap-01`.

## Safety Gates

- Fresh installs do not transfer secrets, media, browser sessions, SQLite databases, or rendered review kits.
- A real bootstrap refuses a blank Upload-Post profile.
- Publish jobs use the one configured local Upload-Post profile; jobs cannot select a different profile.
- Instagram, YouTube, Facebook, and X stay blocked for posting until individually warmed and enabled later.
